# AI 原生企业效率 Agent：需求文档 + 架构设计文档

## 0. 项目目标

我要开发一个 AI 原生企业效率 Agent。

核心场景：

1. 日程管理
2. 费用报销
3. 企业工作流自动化

# 第一部分：产品需求文档 PRD

## 1. 产品定位

本项目是一个面向企业内部使用的 AI Agent 系统。

它不是传统的日程软件，也不是传统的报销软件，而是一个通过自然语言完成企业事务的智能工作台。

核心理念：

> 用 Agent 替代传统表单式软件交互。

用户只需要说出目标，系统自动完成中间流程。

---

## 2. 核心用户

### 2.1 普通员工

主要需求：

- 创建日程
- 修改日程
- 查看日程
- 发起报销
- 查询报销状态
- 补充报销材料
- 理解审批驳回原因

### 2.2 管理者 / 审批人

主要需求：

- 查看待审批报销
- 查看团队日程
- 审批或驳回报销
- 让 AI 总结某个人或某个项目的费用情况

### 2.3 系统管理员

主要需求：

- 配置报销规则
- 配置审批流
- 查看 Agent 执行日志
- 排查任务失败原因
- 管理用户权限

---

## 3. 核心痛点

传统企业系统的问题：

1. 表单太复杂
2. 功能入口太多
3. 用户不知道该填什么
4. 报销经常因为缺字段被驳回
5. 日程修改和冲突检查麻烦
6. 任务失败后日志难以追踪
7. 用户需要理解系统规则，而不是系统理解用户

本项目要解决的问题：

1. 用户用自然语言表达需求
2. AI 自动把自然语言转成结构化任务
3. AI 自动判断缺失字段
4. AI 主动追问用户
5. AI 自动调用工具完成业务操作
6. 高风险操作前必须让用户确认
7. 所有 Agent 推理和工具调用都要有日志

---

## 4. MVP 范围

第一阶段只做最小可行版本，不做太复杂。

# 5.1 Chat UI：聊天入口

用户通过聊天框输入自然语言任务。

# 5.3 Planner：任务规划器

## 功能说明

Planner 负责把用户目标拆成可执行步骤。

Planner 不直接操作数据库，也不直接调用业务逻辑。

Planner 只输出计划。

## Planner 输出格式

```json
{
  "goal": "创建一笔交通费报销草稿",
  "intent": "expense.create",
  "steps": [
    {
      "step_id": "step_1",
      "action": "extract_expense_info",
      "description": "从用户输入中提取报销信息"
    },
    {
      "step_id": "step_2",
      "action": "check_required_fields",
      "description": "检查必填字段是否完整"
    },
    {
      "step_id": "step_3",
      "action": "ask_user_missing_info",
      "description": "如果缺少字段，向用户追问"
    },
    {
      "step_id": "step_4",
      "action": "create_expense_draft",
      "description": "创建报销草稿"
    },
    {
      "step_id": "step_5",
      "action": "wait_user_confirmation",
      "description": "等待用户确认是否提交"
    }
  ]
}
```

# 5.4 Executor：执行器

## 功能说明

Executor 按照 Planner 生成的步骤执行任务。

Executor 不能自己乱改数据库，必须通过 Tool Layer 调用工具。

## 工具调用示例

```ts
await tools.calendar.createEvent(input);
await tools.expense.createDraft(input);
await tools.expense.submit(input);
await tools.ocr.parseInvoice(input);
```

## 执行规则

1. 每执行一步都要写日志
2. 每次工具调用都要保存输入和输出
3. 失败时要记录错误原因
4. 高风险操作必须等待用户确认
5. 不能跳过 Validator

---

# 5.5 Validator：校验器

## 功能说明

Validator 负责检查当前任务是否可以继续执行。

检查内容包括：

1. 必填字段是否完整
2. 日期是否合法
3. 金额是否合法
4. 日程时间是否冲突
5. 报销是否符合规则
6. 是否需要用户确认
7. 是否存在高风险操作

## 报销校验示例

```json
{
  "valid": false,
  "missing_fields": ["amount", "invoice_file", "project"],
  "message": "当前报销缺少金额、发票和项目归属，不能提交。"
}
```

## 日程校验示例

```json
{
  "valid": false,
  "conflicts": [
    {
      "title": "已有会议：项目周会",
      "start_time": "2026-05-15 15:00",
      "end_time": "2026-05-15 16:00"
    }
  ],
  "message": "该时间段已有会议，是否仍然创建？"
}
```

Memory 用来保存当前任务上下文和用户偏好。

只在当前会话中有效。

7. 用户默认成本中心

## 4. Agent Runtime 设计

Agent Runtime 是整个系统核心。

### 核心流程

```text
用户输入
  ↓
创建 Agent Session
  ↓
Intent Router 识别意图
  ↓
Planner 生成执行计划
  ↓
Validator 检查字段和风险
  ↓
如果缺字段：追问用户
  ↓
如果字段完整：Executor 调用工具
  ↓
生成草稿
  ↓
等待用户确认
  ↓
确认后保存或提交
  ↓
记录完整日志
```

## 5. Agent Runtime 伪代码

```ts
class AgentRuntime {
  constructor(
    private intentRouter: IntentRouter,
    private planner: Planner,
    private executor: Executor,
    private validator: Validator,
    private memory: MemoryStore,
    private logger: AgentLogger,
  ) {}

  async handleUserMessage(input: {
    userId: string;
    sessionId: string;
    message: string;
  }) {
    await this.logger.logUserMessage(input);

    const context = await this.memory.getSessionContext(input.sessionId);

    const intentResult = await this.intentRouter.detect({
      message: input.message,
      context,
    });

    await this.logger.logIntentResult(input.sessionId, intentResult);

    const plan = await this.planner.createPlan({
      message: input.message,
      intent: intentResult.intent,
      entities: intentResult.entities,
      context,
    });

    await this.logger.logPlan(input.sessionId, plan);

    const validationResult = await this.validator.validate({
      intent: intentResult.intent,
      entities: intentResult.entities,
      context,
    });

    if (!validationResult.valid) {
      await this.memory.updateSessionContext(input.sessionId, {
        intent: intentResult.intent,
        entities: intentResult.entities,
        missingFields: validationResult.missingFields,
      });

      return {
        type: "missing_fields",
        message: validationResult.message,
        missingFields: validationResult.missingFields,
      };
    }

    const executionResult = await this.executor.execute({
      userId: input.userId,
      sessionId: input.sessionId,
      plan,
      context,
    });

    return executionResult;
  }
}
```

## 6. Tool Layer 设计

所有业务能力都必须封装成工具。

Agent 不能直接操作数据库。

## Tool Interface

```ts
export interface AgentTool<Input = any, Output = any> {
  name: string;
  description: string;
  schema: Record<string, any>;
  execute(input: Input): Promise<Output>;
}
```

# 12. Agent 输出格式要求

所有 LLM 输出都必须尽量结构化。

## Intent Router 输出

```json
{
  "intent": "expense.create",
  "confidence": 0.91,
  "entities": {
    "expenseType": "transport",
    "amount": 128,
    "date": "2026-05-13",
    "description": "打车去机场"
  }
}
```

## Planner 输出

```json
{
  "goal": "创建交通费报销草稿",
  "intent": "expense.create",
  "steps": [
    {
      "stepId": "1",
      "action": "check_required_fields",
      "tool": null
    },
    {
      "stepId": "2",
      "action": "create_expense_draft",
      "tool": "expense.createDraft"
    }
  ]
}
```

## Validator 输出

```json
{
  "valid": false,
  "missingFields": ["invoiceFileIds", "projectId"],
  "riskLevel": "low",
  "requiresConfirmation": false,
  "message": "缺少发票和项目归属。"
}
```

# 13. 权限与安全

## 基础规则

1. 用户只能访问自己的日程和报销
2. Agent 不能绕过权限
3. 高风险操作必须二次确认
4. 所有操作都必须写审计日志
5. 工具调用必须有白名单
6. Agent 不允许直接执行任意代码
7. Agent 不允许直接访问数据库
8. 所有外部 API 调用都要封装成 Tool

---

# 14. 错误处理

## 常见错误

1. LLM 输出不是合法 JSON
2. 意图识别失败
3. 用户输入信息不足
4. 工具调用失败
5. 数据库保存失败
6. 日程时间冲突
7. 报销字段缺失
8. 用户取消确认

## 错误返回格式

```json
{
  "type": "error",
  "code": "TOOL_EXECUTION_FAILED",
  "message": "创建报销草稿失败，请稍后重试。",
  "detail": {
    "toolName": "expense.createDraft",
    "reason": "Database timeout"
  }
}
```
