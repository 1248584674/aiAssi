# CLAUDE.md - AI 企业效率 Agent

> [!IMPORTANT]
> **在开始任何任务前，请先完整阅读并严格遵守项目根目录下的 `AGENTS.md` 文件。**
> AGENTS.md 是项目的最高架构约束，本文件的规则不得与之冲突。

## 0. 最高指令: 全域中文模式
1. **强制中文思维**: 所有的思考过程、内部推理、逻辑分析，**必须且只能**使用中文。
2. **对抗惯性**: 即使参考文件是英文，你的思考和输出也必须立刻切回中文。
3. **产物强制**: 文档、注释、Commit Message 必须用中文。
4. **交互语言**: 与用户的所有对话和解释，全程使用中文。

## 1. 思维引导
**利用原生思维链，在推理时自然覆盖以下要点：**
- **上下文**: 当前文件结构和已有代码是怎样的？技术栈是什么？
- **风险**: 改动会不会破坏现有功能？是否引入了不必要的复杂度？

## 2. 核心理念
- **最小变更**: 只做用户要求的事，不擅自添加功能、重构或"改进"无关代码。
- **简洁至上 (KISS)**: 拒绝过度设计，三行相似代码胜过一个过早的抽象。
- **安全优先**: 不引入安全漏洞（XSS、注入等），发现已有漏洞立即修复。
- **可控范围**: 谨慎对待不可逆操作，影响范围大的改动必须先确认。

## 3. 输出规范
- **保持精简**: 直接给出解决方案和代码，避免冗长的功能解释和寒暄。
- **代码命名**: 变量/函数名使用英文 (PEP8)，严禁拼音。
- **中文注释**: 你生成或修改的代码，必须使用中文添加清晰的注释。
- **遵循架构**: 严格采用 `AGENTS.md` 内描述的技术架构，不得随意引入未声明的外部库。
- **Tailwind CSS**: 本项目使用 Tailwind CSS (CDN) 进行样式开发，允许使用 Tailwind 类名。
- **防御自检**: 输出前请自问："我现在的思考过程和回复是中文吗？"

## 4. 工具使用规范
- **专用工具优先**: 读文件用 Read，编辑用 Edit，搜索用 Grep/Glob，而非 Bash。
- **并行执行**: 无依赖的操作尽量并行调用，提高效率。
- **禁止猜测**: 不确定文件路径或函数名时，先搜索确认，不要猜测。

## 5. Skill 使用规则
- **前端/UI 任务**: 涉及页面设计、样式修改、布局调整、组件开发时，调用 `frontend-design` skill。
- **Agent/LLM 任务**: 涉及 Claude API / Anthropic SDK 的代码时，调用 `claude-api` skill。
- **纯后端/脚本任务**: 不需要调用 UI 相关 skill。

## 6. 前端验证
- **UI/前端改动必验证**: 修改了页面、样式、组件等前端代码后，**必须**启动 dev server 并检查效果。
- **非前端代码**: 纯后端/脚本改动不需要浏览器验证，但应运行相关测试确认正确性。

## 7. 安全性保障
> [!IMPORTANT]
> **安全承诺**
> 1. 测试全程仅连接沙盒数据库，原始 `dev.db` **零写入**
> 2. Agent 不能直接操作数据库，必须通过 Tool Layer
> 3. 高风险操作必须用户二次确认
> 4. 所有 Tool 调用必须记录完整日志
> 5. LLM 输出必须通过 Zod 校验
